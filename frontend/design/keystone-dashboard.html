Policy Rules — by violation count
no-removed-fields
Forbids removal of any field from a response schema without a deprecation window of at least 30 days.
Scope: all APIs Violations: 3 payment-service, inventory-svc, user-api
pii-tag-required
All fields containing personally identifiable information must carry x-pii: true in the OpenAPI spec.
Scope: user-facing APIs Violations: 5 auth-gateway, user-api, profile-service
response-schema-stable
Response schemas marked x-stability: stable must not change their shape — additions are allowed, removals or renames are not.
Scope: stable-only APIs Violations: 2 notification-api, search-index
path-naming-convention
All API paths must follow kebab-case and must not include version prefixes (version goes in the header).
Scope: all APIs Violations: 2 legacy-gateway, reporting-api
required-descriptions
Every endpoint, parameter, and schema property must carry a non-empty description field.
Scope: all APIs Violations: 0 Passing
semantic-versioning
Spec version must follow semver. Breaking changes require a major bump; additions require a minor bump.
Scope: all APIs Violations: 0 Passing
rate-limit-headers
All endpoints must include X-RateLimit-Remaining and X-RateLimit-Reset in their response headers.
Scope: public APIs Violations: 0 Passing
